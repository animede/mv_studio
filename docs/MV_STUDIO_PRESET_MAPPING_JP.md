# Music Video Studio プリセット対応表

最終更新: 2026-04-24

## 目的

この表は、`Music Video Studio` の3つの入口プリセットと、`simple_video` が持つ既存の多数プリセット（`VIDEO_PRESETS`）をどう対応づけるかを整理するためのものです。

重要な前提:

- `Music Video Studio` の3プリセットは **入口 / 制作目的** を表します
- `simple_video` の多数プリセットは **実行パイプライン** を表します
- 既存の `simple_video` プリセットは、単純に1STEPへ分解せず、**1本の実行計画** として扱います
- 新UIの `STEP` は、実行単位というより **介入ポイント / 確認ポイント** として使います

---

## 3層モデル

| 層 | 役割 | 例 |
|---|---|---|
| 入口プリセット | 何を作りたいか | キャラ主導MV / 歌詞重視MV / 既存素材編集MV |
| パイプラインプリセット | どう実行するか | `char_edit_i2i_flf`, `t2i_i2v`, `ext_i2i_i2v_mixed` |
| STEP | どこに介入するか | キャラクタ作成 / 歌詞・構成設計 / シーン画像 / シーン動画 / 完成MV |

## A. `simple_video` 既存プリセット → 新UIでの位置づけ

| `simple_video` ID | 既存プリセット名 | パイプライン種別 | 主な用途 | 対応する入口プリセット | 新UIでの扱い |
|---|---|---|---|---|---|
| `t2i_only` | 画像生成のみ（T2I）-- 動画生成なし | 単機能 | キー画像・ラフ作成 | キャラ主導MV / 歌詞重視MV | `シーン画像` STEPの補助実行 |
| `t2i_i2v` | テキストから初期画像生成し動画生成--シーン切替え動画 | T2I→I2V | テキスト起点の基本動画 | 歌詞重視MV | 軽量な標準パイプライン候補 |
| `t2i_i2v_scene_continuous` | テキストから初期画像生成し動画生成（シーン連続） | T2I→連続I2V | シーン継続を重視した動画 | 歌詞重視MV / キャラ主導MV | 連続系の標準候補 |
| `t2v_i2v` | テキストから直接動画を生成--シーン切替え動画 | T2V中心 | 画像を介さない高速系 | 歌詞重視MV | 上級者向け代替候補 |
| `t2v_i2v_scene_continuous` | テキストから直接動画を生成（シーン連続） | T2V→連続I2V | 連続感の強いテキスト起点動画 | 歌詞重視MV | 上級者向け連続系候補 |
| `ext_i2v` | キー画像から連続動画生成 | I2V連続 | 既存画像から動画を伸ばす | 既存素材編集MV / キャラ主導MV | `シーン動画` 再利用系候補 |
| `ext_i2i_i2v_first` | 画像リファイン後動画生成（連続） | I2I→I2V連続 | キー画像を整えてから連続動画化 | キャラ主導MV / 既存素材編集MV | 画像修正込みの連続系候補 |
| `ext_i2i_i2v_scene_cut` | 画像リファイン後動画生成--シーン切替え動画 | I2I→I2V | シーン単位で作り直しやすい | 既存素材編集MV / 歌詞重視MV | `画像差し替え` と相性が良い |
| `ext_i2i_i2v_mixed` | キー画像参照 動画（I2I + 混在トランジション） | I2I+I2V+FLF混在 | トランジションを自動最適化 | キャラ主導MV / 歌詞重視MV | 高品質な標準候補 |
| `char_i2i_flf` | キャラクター動画（I2I + FLF）--連続長尺動画 | キャラ長尺FLF | 連続長尺・高速 | キャラ主導MV | キャラ系の主力候補 |
| `char_edit_i2i_flf` | キャラクター動画（合成+参照選択）--連続長尺動画 | キャラ合成+長尺FLF | キャラの一貫性を強く保つ | キャラ主導MV | 最有力の主力候補 |
| `char_edit_i2v_scene_cut` | キャラクター動画（合成+参照選択）--シーン切替え動画 | キャラ合成+I2V | シーンごとの差し替えがしやすい | 既存素材編集MV / キャラ主導MV | 編集寄りキャラ候補 |
| `char_edit_i2v_mixed` | キャラクター動画（合成+混在トランジション） | キャラ合成+混在 | シーン長可変・混在トランジション自動化・高い柔軟性 | キャラ主導MV / 既存素材編集MV | 上級者にとって最有力になりやすい高機能候補 |

## B. 新UIの入口プリセット → 推奨パイプライン候補

### 1. キャラ主導MV

| 優先度 | 推奨パイプライン | 理由 |
|---|---|---|
| 第1候補 | `char_edit_i2i_flf` | キャラ一貫性、長尺、速度のバランスが最もよい |
| 第2候補 | `char_edit_i2v_mixed` | シーン長可変や混在トランジション自動化を使えるため、慣れると主力になりやすい |
| 第3候補 | `char_edit_i2v_scene_cut` | シーン単位での差し替え運用がしやすい |
| 補助 | `t2i_only` | キャラ案・キービジュアルの初期作成 |

#### 新UI STEPとの対応

| 新UI STEP | 主に介入する内容 | 実行パイプラインとの関係 |
|---|---|---|
| キャラクタ作成 | 参照画像、キャラ合成、固定タグ | `char_edit_*` 系の前処理・基準入力 |
| シナリオ・世界観作成 | シーン構成、歌詞、演出方針 | 各シーンプロンプトの元データ |
| 音楽作成 | 尺、構成、盛り上がり位置 | 連続動画の長さ配分に反映 |
| シーン画像作成 | シーン画像の確認・再生成 | `I2I` 区間への介入 |
| シーン動画作成 | FLF/I2V の再実行、遷移確認 | パイプラインの中核 |
| 完成MV | 結合、確認、差し替え反映 | 最終出力 |

---

### 2. 歌詞重視MV

| 優先度 | 推奨パイプライン | 理由 |
|---|---|---|
| 第1候補 | `t2i_i2v_scene_continuous` | 歌詞・情景をシーン連続で見せやすい |
| 第2候補 | `ext_i2i_i2v_mixed` | 印象カットと遷移品質を両立しやすい |
| 第3候補 | `t2i_i2v` | 分かりやすい標準構成 |
| 上級代替 | `t2v_i2v_scene_continuous` | 画像工程を減らして直接動画生成したい場合 |

#### 新UI STEPとの対応

| 新UI STEP | 主に介入する内容 | 実行パイプラインとの関係 |
|---|---|---|
| 歌詞・構成設計 | 歌詞、シーン区切り、見せ場設計 | シーンプロンプトの基礎 |
| キャラ・モチーフ整理 | 最低限の見た目統一 | 画のブレ抑制 |
| 歌詞と表現のシーン画像 | キーフレーズごとの静止画調整 | `T2I/I2I` 区間への介入 |
| 歌詞同期動画 | ビート・尺・場面転換調整 | `I2V/T2V/FLF` 区間への介入 |
| MV統合 | 最終構成の微修正 | 完成版出力 |

---

### 3. 既存素材編集MV

| 優先度 | 推奨パイプライン | 理由 |
|---|---|---|
| 第1候補 | `ext_i2i_i2v_scene_cut` | 差し替え範囲を局所化しやすい |
| 第2候補 | `char_edit_i2v_scene_cut` | キャラ合成済み素材の部分修正に向く |
| 第3候補 | `ext_i2v` | 動画だけ差し替えたい場合に軽い |
| 高品質代替 | `char_edit_i2v_mixed` | 差し替え後の遷移品質だけでなく、複雑な構成まで扱いたい場合 |

#### 新UI STEPとの対応

| 新UI STEP | 主に介入する内容 | 実行パイプラインとの関係 |
|---|---|---|
| 既存素材読込 | 既存キャンバス、音源、画像、動画の選択 | パイプラインの入力再構成 |
| 画像差し替え | 一部カットの再生成 | `I2I` 中心 |
| 動画差し替え | クリップの再生成、接続確認 | `I2V/FLF` 中心 |
| 完成MV更新 | 差分反映、版管理 | 最終出力 |

## C. 実装時の扱い方

### 方針1: 入口プリセットごとに既定パイプラインを1つ持つ

最初の実装では、各入口プリセットに `default_pipeline_preset_id` を持たせるのが安全です。

例:

| 入口プリセット | `default_pipeline_preset_id` |
|---|---|
| `character_story_mv` | `char_edit_i2i_flf` |
| `lyrics_focus_mv` | `t2i_i2v_scene_continuous` |
| `edit_rebuild_mv` | `ext_i2i_i2v_scene_cut` |

### 方針2: STEPは「実行単位」ではなく「介入ポイント」

新UIでは、STEPをそのまま `simple_video` の実行単位にしない方がよいです。

- パイプライン実行は `simple_video` プリセット単位
- UI上のSTEPは確認・再入力・差し替え・再実行の入口

### 方針3: 上級者向けにパイプライン切替を許可

各入口プリセットで、必要なら `代替パイプライン` を選べるようにします。

例:

- 標準
- 高品質
- 高速
- 編集向け
- 長尺向け

このとき `simple_video` の内部IDをそのまま見せるのではなく、入口プリセット選択後のユーザー向け名称を別に持たせるのが望ましいです。見出しは `制作フロープラン選択` のように、AIの内部処理よりも制作体験を前面に出す表現が適しています。

例:

| 内部ID | ユーザー向け表示名 |
|---|---|
| `char_edit_i2i_flf` | キャラ一貫性を重視した標準動画制作 |
| `char_edit_i2v_scene_cut` | シーン単位で調整しやすい動画制作 |
| `char_edit_i2v_mixed` | 品質と柔軟性を両立させた高度な動画制作 |

特に `char_edit_i2v_mixed` は、シーン長可変や混在トランジション自動化を含むため複雑な動画に強く、慣れたユーザーでは最も使われる候補になりやすい点に注意します。

## E. 結論

- `simple_video` の多数プリセットは **STEP分解せず、パイプライン資産として活かす**
- `Music Video Studio` の3プリセットは **入口整理に専念する**
- 新UIのSTEPは **介入ポイント / 編集ポイント** として扱う
- 最初は **各入口プリセットに既定パイプラインを1本割り当てる** のが最も安全
